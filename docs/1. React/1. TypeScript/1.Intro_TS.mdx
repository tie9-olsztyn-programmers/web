# Co to jest TypeScript?
- TypeScript (TS) to nadzbiór języka JavaScript, który wprowadza do niego statyczne typowanie i zaawansowane funkcje.
- w czasie kompilacji jest zmieniany na JavaScript

# Dlaczego TypeScript?
Bezpieczeństwo typów:
- **Zapobiega** wielu błędom, które występują w JavaScript, np. wywoływaniu funkcji z nieprawidłowymi argumentami.
- **Skalowalność**: ułatwia zarządzanie dużymi projektami, gdzie typowanie strukturalne pozwala szybko zrozumieć zależności między modułami.
- **Refaktoryzacja**: dzięki typom refaktoryzacja kodu jest mniej ryzykowna. Narzędzia TS automatycznie pokazują, gdzie zmiana jednej części kodu wpłynie na inne.
- **Popularność**: TypeScript jest szeroko stosowany w profesjonalnych projektach i frameworkach, takich jak Angular, NestJS, Next.js, a nawet React.

# Jak działa TypeScript?
- kod w TypeScript: Piszemy kod w plikach `.ts` lub `.tsx`.
- kompilacja do JavaScript: TypeScript kompiluje kod do czystego JavaScript, który jest zgodny z różnymi środowiskami (przeglądarki, Node.js).
- uruchamianie JavaScriptu: Wynikowy kod JavaScript jest wykonywany, więc do działania TypeScriptu nie potrzebne są specjalne środowiska.

## Instalacja
### Linux oparty na debianie
`sudo npm install -g typescript`

## Kompilacja
`tsc nazwaPliku.ts`

## Uruchomienie pliku JavaScriptowego
`node nazwaPliku.js`


# Podsumowanie
- TypeScript to rozszerzenie JavaScriptu, które dodaje statyczne typowanie i ułatwia tworzenie dużych, skalowalnych aplikacji.
- wykrywa błędy na wczesnym etapie i poprawia jakość kodu.
- jest kompatybilny z JavaScript i nowoczesnymi bibliotekami oraz frameworkami.
- używany zarówno w aplikacjach frontendowych (React, Angular), jak i backendowych (Node.js, NestJS).
